# Buddy (NEXUS PROTOCOL) - UI Architecture Spec

## 1. コンセプト・設計哲学
**「思考ノイズをゼロにし、実行のみに集中するための人生のコックピット」**

本UIは、装飾や複雑な画面遷移を徹底的に排除し、処理速度と摩擦ゼロのUXを最優先して設計されている。TOEIC学習、大学編入対策、そして理想の肉体構築といった日々の目標に対して、ユーザーが「迷う時間」を物理的に奪い、ただ「実行と記録」を繰り返すためのシステムである。

* **Color Palette:** 深淵なダークグレー（`#121212`）をベースとし、集中を妨げない低彩度のシアン（`#00E5FF`）とエメラルドグリーンをアクセントに使用。
* **Minimalist Design:** ボーダーやドロップシャドウを極力排除し、余白とタイポグラフィの美しさで情報を整理。

## 2. 技術スタック
* **Framework:** Next.js (App Router)
* **Styling:** Tailwind CSS
* **State Management:** React Hooks (`useState`, `useMemo`)

## 3. コア・コンポーネント仕様

### A. Today's Focus（本日のタスク）
今日実行すべきタスクのみをフィルタリングして表示する。
* **機能:** タスクの確認と1タップでの完了処理。
* **UX:** 完了した瞬間にリストから消去（Done Logへ移動）され、判断コストを最小化。「何を残しているか」だけを可視化する。

### B. Health & Body（肉体・健康記録）
体重と最低限のワークアウトを即座に記録するエリア。
* **機能:** 体重の数値入力（目標80.0kgに対する現在値）と、自重トレーニング（腕立て、スクワット等）のワンタップ記録。
* **UX:** 入力フィールドを最小限に抑え、運動記録はボタンタップのみで完了する。状態は即座にUIへ反映（色変更）される。

### C. Done Log（実績の蓄積）
完了したタスクとワークアウトを時系列で積み上げるエリア。
* **機能:** 完了済みアイテムをタイムライン形式で一覧表示。
* **UX:** 「今日これだけやり切った」という圧倒的な達成感を視覚的に提供。完了時刻とタグをミニマルに表示し、一日の密度を一目で把握可能にする。

### D. Brain Dump（思考ノイズの掃き出し口）
不意に浮かんだアイデアや雑念を一時的に退避させるメモ領域。
* **機能:** 画面下部に常駐する1行のテキスト入力フィールド。
* **UX:** 親指が届きやすい下部配置により、ページ遷移なしで思考を即座にダンプ。入力後はすぐメインタスク（本線）へ意識を戻せる設計。

## 4. 拡張ロードマップ（Next Steps）
プロトタイプから実運用レベルへ引き上げるための実装順序：

1.  **Local Storage Integration**
    * Reactの `useEffect` を用いて、タスク、体重、運動履歴、メモのステートをブラウザのLocalStorageへ永続化する。
    * 日付をキーにしたデータ構造（`YYYY-MM-DD`）へ改修し、日単位でのデータ分離を実現する。
2.  **Mobile UX Optimization**
    * スワイプジェスチャーの導入。モバイル環境でのタスク完了・削除処理をより直感的に高速化する。
3.  **Backend Integration (Supabase)**
    * PostgreSQL + Supabaseによるデータベース連携。
    * 複数端末（PC・スマートフォン）間でのリアルタイム同期と、長期間のデータ蓄積・グラフ化（テレメトリー）の基盤を構築する。
